Pick a domain

The domain is your outbound sending identity. Pick one that *sounds like a service business*, not "youragency.io." Recipients see it in the From line of every cold email.

Buy at Namecheap, Cloudflare Registrar, or Porkbun. .com is the cheapest to maintain reputation; .co and .net work too.

DNS records (copy-paste)

Add these in your registrar's DNS panel. Skipping DKIM/SPF/DMARC is the #1 reason cold emails land in spam.

SPF — authorise your mail server to send on your behalf

Type:    TXT
Host:    @
Value:   v=spf1 include:_spf.google.com ~all

DKIM — proves your mail wasn't tampered with

Your mail provider generates the DKIM key. Paste it as a TXT record on the selector host shown.

Type:    TXT
Host:    google._domainkey          
Value:   v=DKIM1; k=rsa; p=MIGfMA0GCSq...   (paste what Google/Zoho/MP shows)

Per-host instructions: Google Workspace → Admin → Apps → Gmail → Authenticate email. Zoho Mail → Setup → Email Authentication. Microsoft 365 → Security → Email authentication.

DMARC — tells receivers what to do when SPF/DKIM fails

Type:    TXT
Host:    _dmarc
Value:   v=DMARC1; p=none; rua=mailto:dmarc@YOUR-DOMAIN.COM

Start with p=none (monitor only); flip to p=quarantine after week 3 if your reports are clean.

MX — so replies come back to you

Type:    MX
Host:    @
Value:   1 ASPMX.L.GOOGLE.COM       
        10 ALT1.ASPMX.L.GOOGLE.COM
        10 ALT2.ASPMX.L.GOOGLE.COM

Pick a mailbox provider

You need real IMAP + SMTP, not a Gmail alias.

Google Workspace

$7/user/mo. Best deliverability of the three. Branded as your domain.

Steps: workspace.google.com → Add domain → Verify ownership → Create user hello@your-domain.com.

Zoho Mail (free tier)

Free up to 5 users. Long setup time but it works.

Steps: zoho.com/mail → Add domain → Verify TXT record → Enable IMAP.

ForwardEmail.net

Free DNS-level forwarding to your existing Gmail. IMAP not included.

Only recommended if you'll be sending through Amazon SES or a non-IMAP relay.

⚠️ Don't use: (1) a regular Gmail address (you can't get branded From, deliverability tanks), (2) Yahoo/AOL (low reputation in B2B), (3) any "verify your domain later" trial.

Connect the data sources

Both have generous free tiers. Apollo is the better source for B2B prospects with verified decision-maker email; Google Places is better for *local* service businesses (HVAC, plumbing, dentists).

Apollo.io

Sign up at apollo.io → Settings → Integrations → API. Free trail gives 10k credits/mo.

Google Places API (New)

Google Cloud → APIs → Places API (New) → Enable billing → create API key. ~$32 / 1000 calls.

LLM (Groq)

Get a free API key at console.groq.com/keys. The agency app uses it to lightly rewrite each opener so it reads like a person wrote it, and to classify inbound replies as Hot / More info / Not now / etc. If Groq is unreachable or unconfigured, email generation falls back to the pure template and replies fall back to keyword matching — campaigns still send, just less smart.

Open-rate tracking (1×1 pixel)

Embeds an invisible GIF in each cold email so the dashboard can compute open rate per client. Requires the Flask app to be reachable from the public internet (Caddy proxies HTTPS to it on :5000). Disable for compliance-sensitive sends (GDPR-strict EU recipients).
